Front-end

"프론트엔드(front-end)"는 사용자가 웹사이트나 앱을 사용할 때
직접 눈으로 보고, 클릭하거나, 스크롤하면서 사용하는 화면을 만드는 부분입니다.
예를 들어, 웹사이트의 버튼, 메뉴, 이미지, 텍스트 같은 사용자 인터페이스(UI) 요소가 포함됩니다.

프론트엔드 개발은 'HTML', 'CSS', 'JavaScript'와 같은 기술을 사용합니다.
HTML로 뼈대를 만들고, CSS로 디자인을 꾸미고, JavaScript로 움직이는 기능을 추가합니다.

사용자가 입력한 데이터를 처리하거나 서버를 통신하는 백엔드(back-end)와 상호작용합니다.

2. 웹 호스팅

웹 호스팅(Web Hosting)은 웹사이트를 인터넷 상에 공개하고
사람들이 접근할 수 있도록 하기 위해 웹 서버 공간을 제공하는 서비스입니다.

이를 이용하면 개인이나 기업이 자체 서버를 구축할 필요 없이,
제공받은 서버 공간에 웹사이트 파일을 업로드하여 운영할 수 있습니다.

aws, 카페24 등 이러한 서비스를 제공하는 곳은 매우 많지만,
그 중에서도 깃허브는 코드백업, 버전관리와 협업의 툴임과 동시에 웹 호스팅도 무료로 사용이 가능하여
전세계적으로 많은 사람들이 이용하고 있습니다.

깃허브 웹 호스팅

  • 1) 깃허브에 접속 및 로그인
  • 2) repository 생성
  • 3) index.html 파일 업로드
  • 4) Settings - Pages에서 None을 main으로 변경 후 Save
  • 5) 완료되면 주소가 뜨고, 이 주소는 누구나 방문할 수 있게 됨

3. 시맨틱 태그 설정

시멘틱 태그(Semantic tag)는 HTML에서 콘텐츠의 의미를 명확하게 전달하기 위해 사용하는 태그입니다.

시맨틱 태그의 사용은 소스 코드를 구조화하여 코드의 가독성을 높이고, 유지보수를 보다 용이하게 만듭니다.
또한, 검색엔진 상에서 웹사이트 노출 순위를 높이고 더 많은 트래픽을 유도할 수 있습니다.

시맨틱 태그는 콘텐츠의 목적과 역할을 설명합니다. 종류는 다음과 같습니다.

  • 시맨틱 태그의 종류
  • 태그 설정 및 적용

nav, section, footer 시멘틱 태그를 사용하여 항목에 대한 구별을 명확히 합니다.

'nav 태그'는 페이지에서 내비게이션 링크를 제공하는 것을 나타냅니다.
같은 페이지의 다른 부분을 가리키는 링크 또는 전혀 다른 출처로 이동하는 링크일 수도 있습니다.

'section 태그'는 내용의 한 부분을 나타냅니다. 특정 그룹을 묶는 목적으로 쓰입니다.

'footer 태그'는 꼬리말을 나타냅니다. html 파일의 밑부분에 쓰입니다.

5. 느낀 점(회고)

다른 개발 영역의 도전 임베디드 개발자로 지내며, 다른 개발 분야에 대한 호기심은 있었지만, 어디서부터 시작해야 될지 막막해 미루어 왔습니다.

퇴사 후, 취업 준비를 하는 과정에서 다양한 개발 분야의 강의에 대해 알게 되었고 프론트엔드에 대해 본격적으로 배울 수 있게 되었습니다.

처음 시작은 힘들지만, 새로운 도전은 이루었을 때 큰 성취감을 가져다 주었습니다.
개발 지식 확장 및 소프트웨어 활용 능력 향상 HTML, CSS, JavaScript 등 프론트엔드를 대표하는 개념에 대해 알게 되고,
코드를 더 자세히 보기 위해서만 활용했던 'Visual Studio Code'를 'GitHub'와 연동시켜 본격적인 개발 활동을 시작하게 되었습니다.

GitHub가 제공하는 웹 호스팅 기능과 VS Code에서 제공하는 'Live Server'를 활용하여
작성한 내용이 웹 페이지에서 변경되는 것을 확인했을 때, 더 전문적인 개발자가 될 수 있었습니다.
새로운 개발의 재미 부품과 소프트웨어마다 데이터시트와 명령어가 달라 개발환경설정에 많은 어려움을 겪었던 임베디드와 달리,
프론트엔드는 큰 제약없이 노트북만으로 원하는 결과를 확인할 수 있어서 배우는 것이 되게 즐거운 시간이었습니다.
